## 上下文

BS Forum 为 Vue3 + ASP.NET Core 8 单体论坛。用户壳层为 `AppLayout`，管理后台为独立 `AdminLayout`。充值卡兑换、屏蔽列表、卡密库存等后端能力已部分存在，UI 未闭环。本变更聚焦「立刻做」批次，不引入新基础设施。

## 目标 / 非目标

**目标：**
- 降低顶栏噪音，突出搜索/发帖/通知/私信
- 补齐用户卡密兑换与屏蔽管理
- 管理驾驶舱增加同比与简单预警文案
- 管理端可浏览卡密库存
- 统一明显不协调的按钮样式

**非目标：**
- 不做关注动态独立页、新手引导、版主工作台（下一批）
- 不做暗黑主题、A/B、复杂埋点平台
- 不改 JWT/权限模型

## 决策

1. **顶栏资产收入头像菜单**：用 CSS/DOM 重组，避免新路由；窄屏继续隐藏次要信息。
2. **玩法入口下拉**：主导航保留「首页」，增加「玩法」下拉含签到/商城/邀请/转盘，减少一级噪音。
3. **Stats 增量字段**：`AdminStatsDto` 增加昨日对比（yesterdaySignIns 等）与简单 flags；前端展示箭头/百分比，避免新报表服务。
4. **卡密库存**：对接现有 `GET /api/admin/recharge/cards`；若缺分页则最小补 page/pageSize。
5. **屏蔽管理**：设置页新区块调用已有 blocked list / unblock API。

## 风险 / 权衡

- [顶栏改动影响所有页] → 保持视觉 token，仅搬迁信息，不做大改版
- [Stats 查询变重] → 昨日指标用日期范围 Count，与今日查询同量级
- [卡密列表含敏感码] → 仅 Admin 角色；列表可部分掩码显示
